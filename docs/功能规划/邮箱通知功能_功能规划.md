## 一、功能概述

### 1.1 目标
为系统提供邮箱验证和后台开关控制功能，实现：
- 用户注册时邮箱/手机验证
- 商家入驻时邮箱/手机验证
- 商家绑定邮箱接收订单通知
- 后台统一控制各功能开关

### 1.2 业务场景

| 场景 | 触发条件 | 说明 |
|------|---------|------|
| 用户注册 | 后台开启注册功能 | 支持邮箱验证/手机验证/无验证 |
| 商家入驻 | 后台开启商家入驻 | 支持邮箱验证/手机验证 |
| 商家绑定邮箱 | 商家主动绑定 | 用于接收订单通知 |
| 订单通知 | 用户购买文章 | 邮件通知商家 |
| 提现通知 | 商家提现审核 | 邮件通知结果 |

### 1.3 后台开关配置

| 开关名称 | 说明 | 默认值 |
|---------|------|--------|
| 用户注册开关 | 控制是否允许新用户注册 | 开启 |
| 注册验证方式 | 无验证/手机验证/邮箱验证 | 手机验证 |
| 商家入驻开关 | 控制是否允许商家申请入驻 | 开启 |
| 商家入驻验证方式 | 无验证/手机验证/邮箱验证/双验证 | 手机验证 |
| 邮件通知开关 | 控制是否发送邮件通知 | 关闭 |
| 短信通知开关 | 控制是否发送短信通知 | 开启 |

DROP TABLE IF EXISTS `la_email_log`;
CREATE TABLE `la_email_log` (
  `id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT,
  `scene_id` int(10) NOT NULL DEFAULT 0 COMMENT '场景ID',
  `email` varchar(100) NOT NULL DEFAULT '' COMMENT '接收邮箱',
  `title` varchar(200) NOT NULL DEFAULT '' COMMENT '邮件标题',
  `content` text COMMENT '邮件内容',
  `code` varchar(10) NOT NULL DEFAULT '' COMMENT '验证码',
  `send_status` tinyint(1) NOT NULL DEFAULT 0 COMMENT '发送状态:0-发送中,1-成功,2-失败',
  `send_time` int(10) DEFAULT NULL COMMENT '发送时间',
  `is_verify` tinyint(1) NOT NULL DEFAULT 0 COMMENT '是否已验证:0-否,1-是',
  `error_msg` varchar(255) DEFAULT '' COMMENT '错误信息',
  `create_time` int(10) DEFAULT NULL,
  `update_time` int(10) DEFAULT NULL,
  `delete_time` int(10) DEFAULT NULL,
  PRIMARY KEY (`id`),
  KEY `idx_email` (`email`),
  KEY `idx_scene` (`scene_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='邮件发送记录表';
```

## 七、前端页面逻辑

```

    ↓

## 十一、注意事项

1. **安全性**
   - 验证码有效期5分钟
   - 同一邮箱/手机1分钟只能发送1次
   - 验证码使用后立即失效
   - 敏感配置项加密存储

2. **兼容性**
   - 邮箱为选填项
   - 未绑定邮箱不影响正常使用
   - 邮件发送失败不影响业务流程
   - 开关配置有默认值

3. **性能**
   - 邮件发送使用队列异步处理
   - 发送失败记录日志便于排查
   - 配置项使用缓存减少数据库查询

4. **用户体验**
   - 开关关闭时给出友好提示
   - 验证方式切换时表单自动适配
   - 验证码发送倒计时显示
