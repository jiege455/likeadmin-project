## 一、功能概述

### 1.1 目标
将PC端和移动端的装修系统统一，实现**一套装修数据，两端自适应显示**。

### 1.2 业务场景
- 管理员在后台装修页面配置组件
- 组件在移动端按手机屏幕宽度显示
- 同一组件在PC端按PC屏幕宽度自适应显示
- 减少维护成本，避免两端数据不同步

### 1.3 与现有系统关系
- 复用现有 `la_decorate_page` 表结构
- 复用现有组件库（widgets）
- 修改PC端渲染逻辑，使其能读取移动端装修数据

## 三、数据库设计

### 3.1 现有表结构（无需修改）

```sql
-- la_decorate_page 表保持不变
CREATE TABLE `la_decorate_page` (
  `id` int(11) unsigned NOT NULL AUTO_INCREMENT,
  `type` tinyint(2) unsigned NOT NULL DEFAULT 10 COMMENT '页面类型 1=商城首页, 2=个人中心, 3=客服设置, 4=PC首页, 5=系统风格, 10=自定义页面',
  `name` varchar(100) NOT NULL DEFAULT '' COMMENT '页面名称',
  `data` text COMMENT '装修数据',
  `meta` text COMMENT '页面设置',
  `create_time` int(10) unsigned DEFAULT NULL,
  `update_time` int(10) unsigned DEFAULT NULL,
  PRIMARY KEY (`id`)
);
```

### 3.2 数据迁移说明

- 保留 `id=1`（移动端首页）作为统一首页数据
- 保留 `id=2`（个人中心）作为统一个人中心数据
- `id=4`（PC首页）数据可废弃或作为备份
- PC端读取 `id=1` 的数据，按PC布局渲染

## 七、组件自适应方案

### 7.1 组件宽度适配

| 组件 | 移动端 | PC端 |
|------|--------|------|
| 轮播图(banner) | 100%宽度 | 最大1200px居中 |
| 导航(nav) | 5列/行 | 8-10列/行 |
| 资讯(news) | 单列 | 多列网格 |
| 搜索(search) | 全宽 | 固定宽度居中 |
| 用户信息(user-info) | 全宽 | 卡片式居中 |

### 7.2 PC端组件样式覆盖

```scss
// PC端组件自适应样式
.widgets-container {
  max-width: 1200px;
  margin: 0 auto;
  
  // 轮播图
  .widget-banner {
    width: 100%;
    border-radius: 8px;
    overflow: hidden;
  }
  
  // 导航菜单
  .widget-nav {
    .nav-item {
      width: calc(100% / 8); // PC端8列
    }
  }
  
  // 资讯列表
  .widget-news {
    display: grid;
    grid-template-columns: repeat(3, 1fr); // PC端3列
    gap: 20px;
  }
}
```
