## 1. 功能概述

### 目标
为提现账户添加收款码上传功能，用户可以上传支付宝收款码、微信收款码图片。商家/推广员提现时，管理员可以直接扫码打款。

### 业务场景
1. 用户在添加/编辑收款账户时，可上传对应类型的收款码图片
2. 管理员在审核提现申请时，可以查看收款码图片并扫码打款
3. 支持支付宝、微信两种收款码类型

### 与现有系统关系
- 基于 `withdraw_account` 表扩展
- 复用现有文件上传服务
- 与提现审核流程关联

## 3. 数据库设计

### 表结构修改 (la_withdraw_account)

```sql
ALTER TABLE `la_withdraw_account` 
ADD COLUMN `qrcode` varchar(255) DEFAULT '' COMMENT '收款码图片' AFTER `bank_branch`;
```

### 字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| qrcode | varchar(255) | 收款码图片URL，为空表示未上传 |

## 6. 接口规划

### 用户端接口

| 接口路径 | 方法 | 参数 | 返回值 | 说明 |
|---------|------|------|--------|------|
| /withdraw.account/add | POST | qrcode | - | 添加账户时上传收款码 |
| /withdraw.account/edit | POST | qrcode | - | 编辑账户时更新收款码 |
| /withdraw.account/lists | GET | - | lists[].qrcode | 返回收款码URL |
| /withdraw.account/detail | GET | - | qrcode | 返回收款码URL |

## 7. 测试步骤

### 测试1：添加账户上传收款码
1. 进入「添加收款账户」页面
2. 选择「支付宝」类型
3. 填写账号、姓名
4. 上传支付宝收款码图片
5. 点击保存
6. 预期：账户列表显示收款码缩略图

### 测试2：编辑账户更新收款码
1. 进入账户列表，点击编辑
2. 更换收款码图片
3. 保存
4. 预期：收款码已更新

### 测试3：提现审核查看收款码
1. 用户提交提现申请
2. 管理员进入提现审核
3. 查看提现详情
4. 预期：显示收款码大图，可扫码打款
