## 一、功能概述

### 1.1 目标
在现有公共聊天室基础上，新增一对一私聊功能，让用户可以与商家进行私密沟通。

### 1.2 业务场景
| 场景 | 描述 |
|------|------|
| 用户咨询商品 | 用户对商家发布的资料有疑问，需要私下沟通 |
| 预约服务 | 用户需要与商家预约线下服务时间 |
| 投诉反馈 | 用户有问题需要私下与商家沟通解决 |
| 商务合作 | 用户想与商家进行商务合作洽谈 |

### 1.3 与现有系统关系
- 基于现有 WebSocket 服务（GatewayWorker）扩展
- 复用现有 `ChatWindow.vue` 组件，增加私聊模式
- 整合到消息中心模块，新增私聊列表
- 使用现有用户体系和商家体系

## 三、数据库设计

### 3.1 新增私聊会话表 `la_chat_conversation`

```sql
DROP TABLE IF EXISTS `la_chat_conversation`;
CREATE TABLE `la_chat_conversation` (
  `id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `conversation_id` varchar(64) NOT NULL DEFAULT '' COMMENT '会话ID，格式：user_{小ID}_{大ID}',
  `user_id` int(11) UNSIGNED NOT NULL COMMENT '用户ID',
  `target_id` int(11) UNSIGNED NOT NULL COMMENT '对方ID（商家ID或用户ID）',
  `target_type` tinyint(1) NOT NULL DEFAULT 1 COMMENT '对方类型：1-商家 2-用户',
  `last_message` varchar(500) NOT NULL DEFAULT '' COMMENT '最后一条消息内容',
  `last_message_time` int(10) UNSIGNED DEFAULT NULL COMMENT '最后消息时间',
  `unread_count` int(11) NOT NULL DEFAULT 0 COMMENT '未读消息数',
  `is_deleted` tinyint(1) NOT NULL DEFAULT 0 COMMENT '是否删除：0-否 1-是',
  `create_time` int(10) UNSIGNED DEFAULT NULL COMMENT '创建时间',
  `update_time` int(10) UNSIGNED DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_conversation_user` (`conversation_id`, `user_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_target_id` (`target_id`),
  KEY `idx_last_message_time` (`last_message_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='私聊会话表';
```

### 3.2 新增禁言记录表 `la_chat_ban`

```sql
DROP TABLE IF EXISTS `la_chat_ban`;
CREATE TABLE `la_chat_ban` (
  `id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `user_id` int(11) UNSIGNED NOT NULL COMMENT '被禁言用户ID',
  `user_type` tinyint(1) NOT NULL DEFAULT 1 COMMENT '用户类型：1-普通用户 2-商家',
  `ban_type` tinyint(1) NOT NULL DEFAULT 1 COMMENT '禁言类型：1-私聊禁言 2-公共聊天禁言 3-全部禁言',
  `reason` varchar(255) NOT NULL DEFAULT '' COMMENT '禁言原因',
  `admin_id` int(11) UNSIGNED NOT NULL COMMENT '操作管理员ID',
  `expire_time` int(10) UNSIGNED DEFAULT NULL COMMENT '禁言到期时间，NULL表示永久',
  `status` tinyint(1) NOT NULL DEFAULT 1 COMMENT '状态：0-已解除 1-禁言中',
  `create_time` int(10) UNSIGNED DEFAULT NULL COMMENT '创建时间',
  `update_time` int(10) UNSIGNED DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_user_type` (`user_type`),
  KEY `idx_status` (`status`),
  KEY `idx_expire_time` (`expire_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='聊天禁言记录表';
```

### 3.3 扩展现有消息表 `la_chat_message`

消息表已有 `room_id` 字段，私聊消息使用格式：`private_{小ID}_{大ID}`

```sql
-- 为私聊消息添加索引（如果不存在）
ALTER TABLE `la_chat_message` 
ADD INDEX `idx_user_target` (`user_id`, `room_id`);
```

### 3.4 会话ID生成规则

```
规则：private_{小用户ID}_{大用户ID}
示例：用户5与商家2私聊 → conversation_id = private_2_5
好处：双方共享同一个会话ID，便于查询和管理
```

## 九、技术要点

### 9.1 会话ID设计

```
会话ID = 'private_' + min(用户ID, 对方ID) + '_' + max(用户ID, 对方ID)

优点：
1. 双方共享同一会话ID
2. 无需额外查询即可确定房间
3. 避免重复创建会话
```

### 9.2 消息推送机制

```
私聊消息推送流程：
1. 用户A发送消息
2. 服务端保存消息到数据库
3. 服务端根据 conversation_id 查找房间内所有连接
4. 推送消息给房间内所有用户（即用户A和用户B）
```

### 9.3 未读消息处理

```
未读消息计数：
1. 用户B收到消息时，如果不在当前会话页面，unread_count + 1
2. 用户B进入会话页面时，调用 read 接口，unread_count 清零
3. 前端显示红点或数字提示
```

### 9.4 消息安全

```
安全措施：
1. 验证用户是否有权限查看该会话
2. 过滤敏感词汇
3. 防止SQL注入
4. 消息内容转义防XSS
```

## 十、注意事项

1. **会话创建时机**：用户点击私聊时自动创建，无需提前创建
2. **消息隔离**：私聊消息与公共聊天室消息完全隔离
3. **离线消息**：用户离线时消息保存到数据库，上线后可查看
4. **历史消息**：支持分页加载历史消息
5. **会话删除**：删除会话仅删除本地记录，不影响对方和消息记录
6. **商家在线状态**：暂不实现在线状态显示，后续可扩展
