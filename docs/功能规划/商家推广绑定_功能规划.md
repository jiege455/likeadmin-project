# 商家推广绑定与首页定制功能规划

## 1. 功能概述

### 目标
- 商家可以生成专属推广链接/二维码
- 用户通过推广链接注册后自动关注该商家
- 用户首页显示绑定商家的内容
- 用户可以切换关注的其他商家

### 业务场景
```
商家A → 生成推广二维码/链接 → 用户扫码访问 → 注册登录 → 自动关注商家A → 首页显示商家A的内容
                                                  ↓
                                          关注其他商家 → 右上角"切换商家" → 选择已关注的商家
```

### 与现有系统关系
- 基于现有 `la_user` 表、`la_merchant` 表、`la_merchant_follow` 表
- 复用现有注册登录逻辑（已有 `merchant_id` 参数处理）
- 复用现有商家关注功能


## 2. 功能结构

### 模块划分
1. **推广链接模块**
   - 商家生成推广链接/二维码
   - 推广链接包含商家ID参数

2. **注册绑定模块**
   - 解析URL中的商家ID
   - 注册时自动关注商家
   - 设置用户当前显示商家

3. **首页定制模块**
   - 根据当前商家显示对应内容
   - 顶部显示商家信息
   - 显示商家发布的文章

4. **商家切换模块**
   - 右上角"切换商家"按钮
   - 显示已关注商家列表
   - 切换后更新首页内容


## 3. 数据库设计

需要新增字段记录用户当前显示的商家：

```sql

ADD COLUMN `current_merchant_id` int(11) UNSIGNED NOT NULL DEFAULT 0 COMMENT '当前显示的商家ID' AFTER `inviter_id`;
```
