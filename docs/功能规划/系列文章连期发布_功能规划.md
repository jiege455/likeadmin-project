# 系列文章连期发布功能规划

## 1. 功能概述

### 目标
实现系列文章的连期发布功能，让商家可以在同一篇文章中持续更新期次内容，用户购买后可查看当前期及往期历史记录。

### 业务场景
- 商家发布彩票预测类系列文章（如3D每日推荐）
- 每天发布新一期内容，用户购买后可查看最新一期
- 上一期内容作为预览，免费供用户查看
- 用户购买后可查看当前期及之前所有已购买期次

### 与现有系统关系
- 复用现有的系列管理功能（la_article_cate表）
- 复用现有的文章发布流程
- 复用现有的支付购买流程

## 3. 数据库设计

### 3.1 修改文章表 (la_article)

新增字段：

```sql
ALTER TABLE `la_article` ADD COLUMN `prev_issue_content` TEXT NULL COMMENT '上一期内容(免费预览)' AFTER `hidden_content`;
ALTER TABLE `la_article` ADD COLUMN `prev_issue_no` VARCHAR(20) NULL DEFAULT '' COMMENT '上一期期次号' AFTER `prev_issue_content`;
ALTER TABLE `la_article` ADD COLUMN `issue_title` VARCHAR(200) NULL DEFAULT '' COMMENT '期次标题' AFTER `prev_issue_no`;
```

### 3.2 新增期次购买记录表 (la_series_issue_order)

```sql
CREATE TABLE `la_series_issue_order` (
  `id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键',
  `user_id` int(11) UNSIGNED NOT NULL DEFAULT '0' COMMENT '用户ID',
  `series_id` int(11) UNSIGNED NOT NULL DEFAULT '0' COMMENT '系列ID',
  `article_id` int(11) UNSIGNED NOT NULL DEFAULT '0' COMMENT '文章ID',
  `issue_no` varchar(20) NOT NULL DEFAULT '' COMMENT '购买的期次号',
  `order_sn` varchar(32) NOT NULL DEFAULT '' COMMENT '订单编号',
  `pay_price` decimal(10,2) NOT NULL DEFAULT '0.00' COMMENT '支付金额',
  `pay_status` tinyint(1) NOT NULL DEFAULT '0' COMMENT '支付状态:0-未支付,1-已支付',
  `pay_time` int(10) UNSIGNED DEFAULT NULL COMMENT '支付时间',
  `pay_way` tinyint(1) DEFAULT NULL COMMENT '支付方式:1-余额,2-微信,3-支付宝',
  `create_time` int(10) UNSIGNED NOT NULL DEFAULT '0' COMMENT '创建时间',
  `update_time` int(10) UNSIGNED NOT NULL DEFAULT '0' COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_order_sn` (`order_sn`),
  KEY `idx_user_series` (`user_id`, `series_id`),
  KEY `idx_article` (`article_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系列期次购买记录表';
```

## 6. 接口规划

### 6.1 发布文章接口

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/merchant.article/add` | POST | 发布文章（含期次内容） |

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| title | string | 是 | 文章标题 |
| series_id | int | 是 | 系列ID |
| issue_no | string | 是 | 当前期次号 |
| issue_title | string | 否 | 期次标题 |
| prev_issue_content | text | 否 | 上一期内容（免费预览） |
| prev_issue_no | string | 否 | 上一期期次号 |
| hidden_content | text | 是 | 当前期付费内容 |
| price | decimal | 是 | 价格 |

### 6.2 文章详情接口

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/article/detail` | GET | 获取文章详情 |

**返回数据：**
| 字段 | 类型 | 说明 |
|-----|------|------|
| id | int | 文章ID |
| title | string | 文章标题 |
| series_id | int | 系列ID |
| issue_no | string | 当前期次号 |
| prev_issue_content | text | 上一期内容（免费可见） |
| prev_issue_no | string | 上一期期次号 |
| hidden_content | text | 当前期内容（付费后可见） |
| is_buy | bool | 是否已购买当前期 |
| price | decimal | 价格 |

### 6.3 购买期次接口

| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/article/buyIssue` | POST | 购买期次 |

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| article_id | int | 是 | 文章ID |

## 7. 测试步骤

### 7.1 创建系列
1. 进入商家中心 → 系列管理
2. 创建新系列（如：3D每日推荐）
3. 设置系列名称、彩票类型

### 7.2 发布第一期文章
1. 进入发布文章页面
2. 开启系列文章开关
3. 选择所属系列
4. 填写期次号（如：2024001）
5. 填写当前期付费内容
6. 发布

### 7.3 发布第二期文章
1. 进入发布文章页面
2. 开启系列文章开关
3. 选择所属系列
4. 填写期次号（如：2024002）
5. 填写上一期内容（第一期的付费内容，作为预览）
6. 填写当前期付费内容
7. 发布

### 7.4 用户端验证
1. 打开文章详情页
2. 验证上一期内容免费可见
3. 验证当前期内容需要购买
4. 购买后验证当前期内容可见
