## 1. 功能概述

### 1.1 目标
扩展现有的"商家投诉"系统为通用的"举报系统"，支持用户对商家和文章进行举报，方便平台管理劣质内容。

### 1.2 业务场景
- 用户浏览文章时，发现内容虚假、侵权或质量差，可点击举报
- 用户浏览商家主页时，发现商家存在欺诈、虚假宣传等行为，可点击举报
- 平台管理员在后台查看和处理举报信息

### 1.3 与现有系统关系
- 扩展现有 `la_merchant_complaint` 表，增加举报类型字段
- 复用现有后台投诉管理页面，扩展筛选功能
- 复用现有前端投诉页面，扩展支持文章举报

## 3. 数据库设计

### 3.1 表结构修改

#### 扩展 la_merchant_complaint 表

```sql
-- 添加举报类型字段
ALTER TABLE `la_merchant_complaint` 
ADD COLUMN `type` tinyint(1) NOT NULL DEFAULT 1 COMMENT '举报类型:1=商家,2=文章' AFTER `id`,
ADD COLUMN `target_id` int(11) NOT NULL DEFAULT 0 COMMENT '目标ID(商家ID或文章ID)' AFTER `type`,
ADD COLUMN `reason` varchar(100) DEFAULT '' COMMENT '举报原因' AFTER `content`;

-- 添加索引
ALTER TABLE `la_merchant_complaint` 
ADD INDEX `idx_type_target` (`type`, `target_id`);

-- 修改表注释
ALTER TABLE `la_merchant_complaint` COMMENT '用户举报表';
```

### 3.2 字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键ID |
| type | tinyint | 举报类型：1=商家，2=文章 |
| target_id | int | 目标ID（商家ID或文章ID） |
| user_id | int | 举报用户ID |
| merchant_id | int | 商家ID（兼容旧数据） |
| content | text | 举报内容详情 |
| reason | varchar | 举报原因（预设选项） |
| images | text | 图片凭证JSON |
| contact | varchar | 联系方式 |
| status | tinyint | 状态：0=待处理，1=已处理 |
| create_time | int | 创建时间 |
| update_time | int | 更新时间 |
| delete_time | int | 删除时间 |

## 6. 接口规划

### 6.1 用户端接口

#### 提交举报
| 项目 | 说明 |
|------|------|
| 路径 | POST /merchant.complaint/add |
| 权限 | 需登录 |
| 参数 | type(类型), target_id(目标ID), content(内容), images(图片), contact(联系方式), reason(原因) |
| 返回 | { code: 1, msg: '提交成功' } |

#### 举报原因选项
```json
{
  "merchant": ["虚假宣传", "服务态度差", "欺诈行为", "侵权内容", "其他原因"],
  "article": ["内容虚假", "质量低劣", "侵权内容", "违法违规", "其他原因"]
}
```

### 6.2 后台管理接口

#### 举报列表
| 项目 | 说明 |
|------|------|
| 路径 | GET /merchant.complaint/lists |
| 权限 | admin |
| 参数 | type(类型), status(状态), user_id, merchant_id, page_no, page_size |
| 返回 | { lists: [], count: 0 } |

#### 处理举报
| 项目 | 说明 |
|------|------|
| 路径 | POST /merchant.complaint/handle |
| 权限 | admin |
| 参数 | id |
| 返回 | { code: 1, msg: '操作成功' } |

## 7. 测试步骤

### 7.1 文章举报测试
1. 用户登录前端
2. 进入任意文章详情页
3. 点击底部"举报"按钮
4. 选择举报原因，填写详细内容
5. 提交举报
6. 后台查看举报记录，确认类型为"文章"
7. 点击"处理"，确认状态变更

### 7.2 商家举报测试
1. 用户登录前端
2. 进入任意商家主页
3. 点击"举报"按钮
4. 选择举报原因，填写详细内容
5. 提交举报
6. 后台查看举报记录，确认类型为"商家"
7. 点击"处理"，确认状态变更

### 7.3 后台筛选测试
1. 登录后台管理系统
2. 进入"商家管理 > 投诉管理"
3. 筛选"举报类型=文章"，确认只显示文章举报
4. 筛选"举报类型=商家"，确认只显示商家举报
5. 筛选"处理状态=待处理"，确认只显示待处理记录
