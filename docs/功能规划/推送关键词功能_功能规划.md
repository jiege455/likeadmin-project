# 推送关键词功能规划

## 1. 功能概述

### 1.1 目标
为用户关注的商家设置推送关键词，当商家发布包含这些关键词的文章时，系统自动推送通知给用户。

### 1.2 业务场景
- 用户关注了"数码科技"商家，设置关键词"iPhone"
- 当该商家发布标题包含"iPhone"的文章时
- 用户收到推送通知，第一时间获取感兴趣的内容

### 1.3 与现有系统关系
- 依赖 `la_user_follow_merchant` 关注表（已有 `is_push` 字段）
- 依赖 `la_notice_record` 通知记录表
- 依赖现有 WebSocket 服务实现实时推送

---

## 2. H5 推送方案分析

### 2.1 H5 推送的可行性

| 推送方式 | 可行性 | 说明 |
|---------|--------|------|
| WebSocket 实时推送 | ✅ 支持 | 系统已有 WebSocket，用户在线时可实时收到 |
| 站内消息 | ✅ 支持 | 存入 NoticeRecord，用户刷新页面可查看 |
| 邮件通知 | ✅ 支持 | 系统已有邮件通知功能 |
| 浏览器推送 (Web Push) | ⚠️ 部分支持 | 需要用户授权，部分浏览器支持 |
| 小程序订阅消息 | ✅ 支持 | 小程序端可使用订阅消息 |

### 2.2 推荐方案
**H5 端采用组合方案：**
1. **在线用户**：WebSocket 实时推送 + 页面内弹窗提示
2. **离线用户**：站内消息存储 + 邮件通知（可选）
3. **小程序端**：订阅消息推送（需用户订阅）

## 4. 数据库设计

### 4.1 新增表：用户推送关键词表

```sql
-- 用户推送关键词表
DROP TABLE IF EXISTS `la_user_push_keyword`;
CREATE TABLE `la_user_push_keyword` (
  `id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT,
  `user_id` int(11) UNSIGNED NOT NULL COMMENT '用户ID',
  `merchant_id` int(11) UNSIGNED NOT NULL COMMENT '商家ID',
  `keyword` varchar(50) NOT NULL COMMENT '关键词',
  `is_enable` tinyint(1) NOT NULL DEFAULT 1 COMMENT '是否启用:1-启用,0-禁用',
  `create_time` int(10) DEFAULT NULL COMMENT '创建时间',
  `update_time` int(10) DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_user_merchant` (`user_id`, `merchant_id`),
  KEY `idx_keyword` (`keyword`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户推送关键词表';
```

### 4.2 新增表：推送消息记录表

```sql
-- 推送消息记录表
DROP TABLE IF EXISTS `la_push_message`;
CREATE TABLE `la_push_message` (
  `id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT,
  `user_id` int(11) UNSIGNED NOT NULL COMMENT '用户ID',
  `merchant_id` int(11) UNSIGNED NOT NULL COMMENT '商家ID',
  `article_id` int(11) UNSIGNED NOT NULL COMMENT '文章ID',
  `keyword` varchar(50) NOT NULL COMMENT '匹配的关键词',
  `title` varchar(200) NOT NULL COMMENT '推送标题',
  `content` varchar(500) DEFAULT '' COMMENT '推送内容',
  `is_read` tinyint(1) NOT NULL DEFAULT 0 COMMENT '是否已读:0-未读,1-已读',
  `push_type` tinyint(1) NOT NULL DEFAULT 1 COMMENT '推送方式:1-站内消息,2-WebSocket,3-邮件',
  `create_time` int(10) DEFAULT NULL COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_merchant_id` (`merchant_id`),
  KEY `idx_article_id` (`article_id`),
  KEY `idx_is_read` (`is_read`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='推送消息记录表';
```

### 4.3 修改现有表：用户关注商户表

```sql
-- 为 la_user_follow_merchant 表添加推送相关字段
ALTER TABLE `la_user_follow_merchant` 
ADD COLUMN `push_keywords` text DEFAULT NULL COMMENT '推送关键词JSON数组' AFTER `is_push`;
```

| `/api/push_keyword/toggle` | POST | 启用/禁用关键词 | 登录用户 |

#### 添加关键词接口详情
```
POST /api/push_keyword/add
请求参数：
{
    "merchant_id": 1,      // 商家ID（必填）
    "keyword": "iPhone"    // 关键词（必填，最多20字符）
}

返回示例：
{
    "code": 1,
    "msg": "添加成功",
    "data": {
        "id": 1
    }
}
```

## 8. 核心流程

### 8.1 文章发布推送流程

```
商家发布文章
    ↓
ArticlePublishListener 监听
    ↓
查询所有关注该商家的用户
    ↓
遍历用户的推送关键词
    ↓
匹配文章标题/内容
    ↓
匹配成功 → 创建推送记录
    ↓
用户在线？ → WebSocket 实时推送
    ↓
用户离线？ → 存入站内消息（下次登录可见）
```

### 8.2 关键词匹配规则

1. **精确匹配**：关键词与文章标题/内容完全匹配
2. **模糊匹配**：关键词包含在文章标题/内容中
3. **多关键词**：用户可设置多个关键词，任一匹配即推送
4. **去重处理**：同一文章对同一用户只推送一次

## 11. 注意事项

1. **性能考虑**：文章发布时批量查询用户关键词，避免 N+1 查询
2. **推送频率**：限制单个用户每日推送次数，避免骚扰
3. **关键词数量**：每个用户每个商家最多设置 10 个关键词
4. **隐私保护**：推送消息不包含敏感信息
5. **兼容性**：H5 端 WebSocket 断线重连机制
