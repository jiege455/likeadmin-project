## 一、功能概述

### 1.1 目标
为系统登录页面添加滑块验证码功能，防止暴力破解、恶意登录和短信轰炸，提升系统安全性。

### 1.2 业务场景
- 后台管理员登录：防止暴力破解管理员账号
- 前端用户登录：防止恶意登录和短信轰炸
- 发送短信验证码前：防止短信接口被恶意调用

### 1.3 与现有系统关系
- 后台登录页面：`admin/src/views/account/login.vue`
- 前端登录页面：`uniapp/src/pages/login/login.vue`
- 后端登录控制器：`server/app/adminapi/controller/LoginController.php` 和 `server/app/api/controller/LoginController.php`

---

## 二、技术方案

### 2.1 实现方案
- **后端**：自定义 `SliderCaptcha` 类，使用 PHP GD 库生成滑块验证码
- **前端**：自定义滑块验证码组件，支持 PC 端和移动端

### 2.2 验证码类型
采用**滑块拼图验证**，用户体验好，安全性高。

## 七、接口规划

### 7.1 获取验证码图片

| 项目 | 说明 |
|------|------|
| 路径 | `/captcha/get` |
| 方法 | GET |
| 权限 | 无需登录 |
| 参数 | 无 |
| 返回 | `{ "bg_img": "base64背景图", "slider_img": "base64滑块图", "key": "验证key", "y": "滑块高度" }` |

### 7.2 验证滑块

| 项目 | 说明 |
|------|------|
| 路径 | `/captcha/check` |
| 方法 | POST |
| 权限 | 无需登录 |
| 参数 | `{ "key": "验证key", "x": "滑动距离" }` |
| 返回 | `{ "verified": true/false }` |

## 八、测试步骤

### 8.1 后台管理登录测试
1. 访问后台登录页面
2. 输入账号密码
3. 点击登录，弹出滑块验证码
4. 拖动滑块完成验证
5. 验证成功后自动登录
6. 验证失败提示重新验证

### 8.2 前端用户登录测试
1. 访问前端登录页面
2. 选择账号密码登录
3. 输入账号密码
4. 点击登录，弹出滑块验证码
5. 拖动滑块完成验证
6. 验证成功后自动登录

### 8.3 安全性测试
1. 不拖动滑块直接登录 → 应失败
2. 滑块位置错误 → 应失败并刷新验证码
3. 连续多次失败 → 正常处理

## 十、注意事项

1. 验证码使用缓存存储，确保缓存服务正常
2. 前端组件需要适配移动端触摸事件
3. 验证码有效期设置为 5 分钟
4. 验证成功后立即清除缓存中的验证码
