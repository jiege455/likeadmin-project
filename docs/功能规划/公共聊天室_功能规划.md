## 一、功能概述

### 1.1 目标
在消息中心添加公共聊天室功能，让所有用户可以在公共频道实时聊天交流。

### 1.2 业务场景
- 用户进入消息中心，可以看到"公共聊天室"入口
- 点击进入后，可以实时看到其他用户发送的消息
- 用户可以发送文字消息，与其他用户实时互动
- 消息会保存到数据库，新用户进入可以看到历史消息

### 1.3 与现有系统关系
- 基于现有的 `ChatWindow.vue` 组件进行完善
- 整合到现有的消息中心模块
- 使用现有用户体系（用户头像、昵称）

## 二、技术方案选择

### 2.1 WebSocket方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| GatewayWorker | PHP原生、易部署、宝塔兼容好 | 需要命令行启动 | ⭐⭐⭐⭐⭐ |
| Swoole | 高性能、功能强大 | 需要安装扩展、配置复杂 | ⭐⭐⭐ |
| Node.js Socket.io | 成熟稳定 | 需要额外Node环境 | ⭐⭐ |

### 2.2 推荐方案：GatewayWorker
- 基于Workerman，纯PHP实现
- 宝塔面板完美支持
- 安装简单，只需 `composer require`
- 支持分布式部署

## 四、数据库设计

### 4.1 聊天消息表 `la_chat_message`

```sql
CREATE TABLE `la_chat_message` (
  `id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `room_id` varchar(50) NOT NULL DEFAULT 'public' COMMENT '聊天室ID，public为公共频道',
  `user_id` int(11) UNSIGNED NOT NULL COMMENT '发送用户ID',
  `nickname` varchar(100) NOT NULL DEFAULT '' COMMENT '用户昵称',
  `avatar` varchar(255) NOT NULL DEFAULT '' COMMENT '用户头像',
  `content` text NOT NULL COMMENT '消息内容',
  `msg_type` tinyint(1) NOT NULL DEFAULT 1 COMMENT '消息类型：1-文字 2-图片 3-系统消息',
  `is_deleted` tinyint(1) NOT NULL DEFAULT 0 COMMENT '是否删除：0-否 1-是',
  `create_time` int(10) UNSIGNED DEFAULT NULL COMMENT '创建时间',
  `update_time` int(10) UNSIGNED DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  KEY `idx_room_id` (`room_id`),
  KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='聊天消息表';
```

### 4.2 聊天室配置表 `la_chat_room`

```sql
CREATE TABLE `la_chat_room` (
  `id` int(11) UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `name` varchar(100) NOT NULL DEFAULT '' COMMENT '聊天室名称',
  `room_id` varchar(50) NOT NULL DEFAULT '' COMMENT '聊天室ID',
  `description` varchar(255) DEFAULT '' COMMENT '聊天室描述',
  `max_users` int(11) NOT NULL DEFAULT 1000 COMMENT '最大用户数',
  `is_public` tinyint(1) NOT NULL DEFAULT 1 COMMENT '是否公开：0-否 1-是',
  `status` tinyint(1) NOT NULL DEFAULT 1 COMMENT '状态：0-禁用 1-启用',
  `create_time` int(10) UNSIGNED DEFAULT NULL COMMENT '创建时间',
  `update_time` int(10) UNSIGNED DEFAULT NULL COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_room_id` (`room_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='聊天室配置表';
```

## 十、宝塔面板部署说明

### 10.1 PHP配置
1. 进入宝塔面板 → 软件商店 → PHP → 设置
2. 安装扩展：`pcntl`, `posix`
3. 禁用函数中删除：`putenv`, `pcntl_signal_dispatch`, `pcntl_wait`, `pcntl_signal`, `pcntl_alarm`, `pcntl_fork`

### 10.2 端口开放
1. 宝塔面板 → 安全 → 添加端口规则
2. 开放端口：8282（WebSocket端口）

### 10.3 启动WebSocket服务
```bash
# 进入项目目录
cd /www/wwwroot/你的网站目录/server

# 启动服务（调试模式）
php websocket/start.php start

# 后台运行（生产环境）
php websocket/start.php start -d

# 停止服务
php websocket/start.php stop

# 重启服务
php websocket/start.php restart
```

### 10.4 设置开机自启（可选）
宝塔面板 → 计划任务 → 添加任务：
- 任务类型：Shell脚本
- 执行周期：开机时
- 脚本内容：
```bash
#!/bin/bash
cd /www/wwwroot/你的网站目录/server
php websocket/start.php start -d
```

## 十一、注意事项

1. **Windows开发环境限制**：Workerman在Windows下不支持后台运行，仅支持调试模式
2. **生产环境**：建议使用Linux服务器部署
3. **消息安全**：需要对消息内容进行过滤，防止XSS攻击
4. **性能优化**：消息分页加载，避免一次性加载过多历史消息
5. **用户隐私**：敏感信息脱敏处理
